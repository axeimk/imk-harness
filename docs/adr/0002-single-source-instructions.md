# ADR-0002: 指示ファイルの単一ソース生成

- ステータス: 承認済み
- 日付: 2026-07-12

## コンテキスト

Claude Code は CLAUDE.md、Codex は AGENTS.md を常駐指示として読む。内容はほぼ共通に
したいが、ファイルを 2 つ手で管理すると必ず乖離する。また Cursor など対応ツールが
増える可能性もある。

## 決定

**`shared/instructions/*.md` を唯一の原本とし、`build.sh` が連結して
`claude/CLAUDE.md` と `codex/AGENTS.md` を生成する。**

- 生成物の先頭に「GENERATED — 編集は shared/ で行うこと」を明記する
- ツール別の差分が必要になった場合は `build.sh` 内で分岐する（過去に Codex 向け追記を
  末尾に付けていた時期があり、この方式で差分を吸収できることは実証済み）

## 検討した代替案

- **CLAUDE.md の `@import` で共通部を参照** — Claude Code 専用機能で AGENTS.md 側
  （Codex）が解釈できない
- **手動で 2 ファイル管理** — 乖離が不可避
- **同一ファイルへの symlink 共有** — ツール別差分を出す手段がない

## 結果

- 編集場所が 1 箇所に定まり、対応ツールの追加は build.sh への出力追加で済む
- 生成ステップが挟まるため、編集の反映には `./install.sh` の再実行が必要（ADR-0004 以降）
