# ADR-0006: ユーザーの持ち物に触れない非破壊方針

- ステータス: 承認済み
- 日付: 2026-07-12

## コンテキスト

アップデート（install.sh 再実行）ではツールの選択変更や削除済みスキルの掃除が発生し、
アンインストールでは配置物の除去が発生する。このとき「ハーネスが置いたもの」と
「ユーザーが自分で置いたもの」を確実に区別しないと、ユーザーのスキルや設定を壊しうる。

## 決定

**「ハーネス由来」と機械的に判定できるものだけを変更・削除の対象とし、それ以外には触れない。**

- **ハーネス由来の判定**: symlink 先が「このリポジトリ配下」であること
  （リンク切れの場合のみ、パスに `imk-harness` を含めば旧残骸として対象に含める）。
  ユーザーが自分で置いたスキル・ファイルは定義上この判定に一致しない
- **コピーで配置するファイル**（settings.json / config.toml）は `copy_if_absent` —
  既存があれば上書きせず、アンインストールでも削除しない（インストール後に編集されて
  いる可能性があるため）。取り込みや削除はサマリで案内し、ユーザーに委ねる
- **既存の実ファイルを置き換える場合**は必ず `.bak.<timestamp>` に退避し、
  除去時には最新のバックアップを復元する
- CLAUDE.md / AGENTS.md の管理ブロック外のテキストは常に保持する（ADR-0004）

## 検討した代替案

- **マニフェストファイル方式**（配置したパスの台帳を持つ） — 台帳とファイルシステムの
  不整合（手動削除・移動）に弱い。symlink 先による判定は状態を持たず自己修復的
- **設定ファイルも symlink / ブロック管理にして追従させる** — settings.json は
  ツール自身も書き換えるファイルであり、ハーネスが所有権を主張すると衝突する

## 結果

- アップデート・アンインストールを何度実行しても、ユーザーの持ち物は壊れない
- 代償として settings.json / config.toml はアップデートに追従せず、手動マージが必要
  （サマリで diff コマンドを案内する）
- リポジトリを移動すると symlink 先の判定が外れるため、移動前に uninstall する運用とする
  （README のエッジケース表に記載）
