# ADR-0004: CLAUDE.md / AGENTS.md は管理ブロック方式

- ステータス: 承認済み
- 日付: 2026-07-12

## コンテキスト

当初 `~/.claude/CLAUDE.md` と `~/.codex/AGENTS.md` はリポジトリの生成物への symlink
だった。この方式はリポジトリ側の編集が即反映される反面、**ユーザーがホーム側のファイルに
独自のテキストを追記できない**（symlink 先は build.sh の生成物なので、直接編集しても
次の生成で消える）という問題があった。

## 決定

**symlink をやめ、実ファイルに ASCII マーカーで囲んだ「管理ブロック」を書き込む方式にする。**

```markdown
（ここはユーザーの自由編集エリア。更新・アンインストールでも保持される）

<!-- >>> imk-harness:begin >>> -->
（ハーネスが生成した共通指示。install.sh 再実行でこの中だけ更新される）
<!-- <<< imk-harness:end <<< -->
```

- 既存ファイルがある場合は末尾にブロックを**追記**し、既存内容には触れない
- 旧 symlink からは install.sh が自動移行する（最新の `.bak.*` があればその内容を
  土台に引き継ぐ）
- アンインストールはブロックのみ除去。ブロックしかないファイルは削除し、
  バックアップがあれば復元する
- ブロックの編集は awk + ASCII マーカーで行う（macOS 標準 bash 3.2 / sed の
  多バイト文字問題を回避するため、マーカーに ASCII 以外を使わない）

## 検討した代替案

- **symlink + リポジトリ内 `local/`（gitignore）上書きディレクトリ** — 単一ソースの思想と
  即時反映は保てるが、「ホームのファイルを直接編集したら消える」という罠が残る
- **CLAUDE.md の `@import`** — Codex（AGENTS.md）が解釈できず、挙動が非対称になる

## 結果

- ユーザーは `~/.claude/CLAUDE.md` を直感どおり編集でき、追記が失われない
- 「build.sh だけで即反映」は失われ、共通指示の反映には `./install.sh` の再実行が必要
- dotfiles 管理ツールで実績のある定番手法（conda init 等と同型）なので挙動が予測しやすい
