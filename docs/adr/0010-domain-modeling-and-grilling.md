# ADR-0010: ユビキタス言語と設計記録の規律を mattpocock/skills から輸入する（意図的な逸脱つき）

- ステータス: 承認済み
- 日付: 2026-07-13

## コンテキスト

[mattpocock/skills](https://github.com/mattpocock/skills) のユビキタス言語の考え方
（CONTEXT.md 用語集、grill-with-docs、軽量 ADR）を汎用層に取り入れたい。
本家の構成をそのまま移すか、このハーネスの構造に合わせて変形するかが論点だった。

本家の現状: ワンショット抽出の ubiquitous-language スキルは非推奨化済みで、
後継は「用語が確定した瞬間に CONTEXT.md を更新する」domain-modeling スキル。
インタビュー技法は grilling プリミティブ + ラッパー 2 本（grill-me / grill-with-docs）の
3 本構成で、grill-with-docs の本文は「grilling を domain-modeling とともに実行せよ」の 1 行。

## 決定

**domain-modeling と grilling の 2 スキルとして導入する。**本家からの意図的な逸脱は 4 つ:

1. **ワンショット抽出は採らない** — 会話から用語を一括収集して用語集を生成する方式は
   本家自身が非推奨化した。継続維持の規律だけを輸入する
2. **ADR の 3 条件のうち「後戻りが難しい」を「覆す・蒸し返すコストが実際に大きい」に
   拡張する** — 規約・方針の決定はテキスト変更だけで覆せるが、記録が無いと同じ検討・
   議論が繰り返される。既存の ADR-0007〜0009 はまさにこの型で、原文の条件を文字どおり
   適用するとすべて落ちる。本家も「却下が非自明な代替案は記録せよ」の条項で実質的に
   同じ緩和をしている
3. **ADR テンプレートは軽量形式（タイトル + 1〜3 文）をデフォルトにし、任意セクションの
   見出し語彙だけ本リポジトリの既存 ADR に固定する** — 重いテンプレートは記録を抑制し、
   エージェントは空セクションを埋め草で埋める。語彙を固定しておけば、軽量 ADR が育つと
   フル形式に収束する（別形式にならない）
4. **grill 系は 3 本構成でなく grilling 1 本に畳む** — ラッパー（grill-with-docs）の
   仕事は grilling と domain-modeling の合成だが、このハーネスでは domain-modeling が
   自動発火し、指示層にも用語集の規約が常駐するため、合成は構造上勝手に起きる。
   スキル 3 本分のメタデータ常駐コストと、スキル間参照（本文 1 行のラッパー）の
   3 ツール互換リスクを避ける。記録が不要な場合はラッパーの使い分けでなく、
   起動文の `--no-docs`（散文で解釈する。引数展開は共有スキルで禁止のため）で抑制する

## 検討した代替案

- **本家 3 本構成（grilling / grill-me / grill-with-docs）の移植** — 決定 4 の理由で却下
- **ADR 条件の原文維持（後戻りが難しい）** — 既存 ADR-0007〜0009 が全部落ちる基準は
  このリポジトリの実運用と矛盾する
- **ADR テンプレートを本リポジトリのフル形式にする** — 記録の抑制と埋め草を招く。
  フル形式が要るプロジェクトでは既存 ADR の形式が優先される（スキルに明記済み）ので、
  デフォルトを重くする必要がない

## 結果

- `shared/skills/domain-modeling/`（CONTEXT.md 用語集 + 軽量 ADR の規律）と
  `shared/skills/grilling/`（徹底インタビュー。domain-modeling への依存は soft —
  無くても成立する）を提供
- `check-skills.sh` の内部参照検査を「ADR 番号・docs/adr へのリンク」に限定
  （展開先プロジェクト自身の置き場を散文で案内する記述は正当なので許可）
- ドッグフーディングとして本リポジトリのルートに `CONTEXT.md` を追加
- 本家の今後の再編への追従義務は負わない。取り込み方を見直すときは本 ADR を更新する
