# 作業の進め方

- コードを変更したら、報告の前にテスト・lint を回す。コマンドはプロジェクトの CLAUDE.md / AGENTS.md の「コマンド」節を参照する（hooks で自動実行される場合はその結果を確認する）。
- あわせて、プロジェクトの verify スキルのレシピに従ってアプリを実際に起動・操作し、変更が動くことを確認する。実行タイミングはコミット前（コミットしない作業では完了報告の前）に 1 回でよく、コード変更のたびに繰り返す必要はない。verify スキルが無ければ既知の方法で動作確認する。
- 上記 2 項は「プログラムの動作に影響する変更」が対象。ドキュメント・エディタ設定（VS Code 設定等）・コメントのみの変更では、テスト・lint・アプリの起動確認は行わない（変更したファイル自体の構文チェックやフォーマッタがあればそれだけ回す）。
- 検証結果は成否をそのまま報告する。テストが失敗したら失敗と言う。伏せたり言い換えたりしない。
- 同種の作業を繰り返していると気づいたら、スキル化を提案する。
- 破壊的な操作（削除、履歴の書き換え、外部への送信）は実行前に確認を取る。
