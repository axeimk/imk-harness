# プロジェクト特化層の育て方（規約）

各プロジェクトには特化層（CLAUDE.md / AGENTS.md、verify スキルなど）を持たせ、プロジェクト固有の知識はそこに置く。

- 特化層の整備（ブートストラップ、不足しているものの提案、採否の記録）は harness-check スキルが担う。ユーザーが明示的に起動する。
- テスト・lint・ビルドのコマンドはプロジェクト CLAUDE.md / AGENTS.md の「コマンド」節に記録する。機械的に強制したい検査（編集後の lint など）は各ツールの hooks に設定する。
- verify スキル（またはそれに相当する手順書）には、そのプロジェクトを実際にビルド・起動・操作して変更を確認するレシピを記録する（テストコマンドの羅列にしない）。
- プロジェクトに `CONTEXT.md`（用語集）があれば、その語彙で会話・命名する。用語のゆらぎに気づいたら追記を、記録に値する設計上の決定が確定したら ADR への記録を提案する（domain-modeling スキル）。
- プロジェクト固有の知識（コマンド、アーキテクチャ、触ってはいけない場所）を汎用層に書かない。
- 2〜3 プロジェクトで同じ指示・スキルを書いたと気づいたら、汎用層（imk-harness リポジトリ）への昇格を提案する。
