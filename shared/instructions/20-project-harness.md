# プロジェクトスコープの育て方（規約）

各プロジェクトのリポジトリにはプロジェクトスコープのハーネス（CLAUDE.md / AGENTS.md、verify スキルなど）を持たせ、プロジェクト固有の知識はそこに置く。

- プロジェクトスコープの整備（ブートストラップ、不足しているものの提案、採否の記録）は harness-check スキルが担う。ユーザーが明示的に起動する。
- 作業の進め方（テスト・lint の回し方、動作確認、報告・確認のルール）はユーザースコープでは定めない。プロジェクト CLAUDE.md / AGENTS.md の「作業の進め方」節に従う。
- テスト・lint・ビルドのコマンドはプロジェクト CLAUDE.md / AGENTS.md の「コマンド」節に記録する。機械的に強制したい検査（編集後の lint など）は各ツールの hooks に設定する。
- verify スキル（またはそれに相当する手順書）には、そのプロジェクトを実際にビルド・起動・操作して変更を確認するレシピを記録する（テストコマンドの羅列にしない）。
- プロジェクトに `CONTEXT.md`（用語集）があれば、その語彙で会話・命名する。用語のゆらぎに気づいたら追記を、記録に値する設計上の決定が確定したら ADR への記録を提案する（domain-modeling スキル）。
- プロジェクト固有の知識（コマンド、アーキテクチャ、触ってはいけない場所）をユーザースコープ（`~/.claude/` などホーム側の設定・指示）に書かない。
- 同種の作業を繰り返していると気づいたら、スキル化を提案する（スキルはプロジェクトスコープに置く）。
