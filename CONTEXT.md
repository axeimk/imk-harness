# imk-harness 用語集

コーディングエージェント用の汎用ハーネス（ユーザースコープの設定・知識・規約一式）を管理するリポジトリの用語集。

## 用語

**ユーザースコープ** (`user scope`):
ホーム側（`~/.claude/` `~/.codex/` `~/.agents/`）に置かれ、全プロジェクトで読み込まれるハーネス資産。このリポジトリが原本を管理し、展開する。
_避ける_: 汎用層、ユーザー層、共通層（旧称。2026-07-19 に改称）

**プロジェクトスコープ** (`project scope`):
各プロジェクトのリポジトリ内に置くハーネス資産（CLAUDE.md / AGENTS.md、verify スキルなど）。プロジェクト固有の知識はすべてここに置く。
_避ける_: 特化層、プロジェクト特化層、プロジェクト層、個別層（旧称。2026-07-19 に改称）

**原本** (`source`):
編集してよい唯一のファイル群（`shared/instructions/*.md`、`shared/skills/*/`）。ここから生成物が作られる。

**生成物** (`generated output`):
`build.sh` が原本から作るファイル（`claude/CLAUDE.md`、`codex/AGENTS.md`）。直接編集禁止。

**管理ブロック** (`managed block`):
ホーム側の CLAUDE.md / AGENTS.md 内でハーネスが所有する、ASCII マーカーで囲まれた区間。区間外はユーザーの自由編集エリアで、絶対に変更しない。
_避ける_: 管理領域、マーカーブロック

**2 段階実行** (`plan-confirm-apply`):
install/uninstall が同一の適用処理を DRYRUN（プラン収集のみ）→ 確認 → 本実行の順で 2 回走らせる方式。
_避ける_: ドライラン方式（DRYRUN は 2 段階実行の前半を指す別概念）

**共有スキル** (`shared skill`):
`shared/skills/` に置かれ、symlink で 3 ツール（Claude Code / Codex / Cursor）に配布されるスキル。どのツールで読まれても本文が成立するように書く。

**HARNESS.md** (`harness record`):
プロジェクトルートに置く、ハーネス提案の採否を記録するファイル。「使わない」と記録された項目は再提案しない。
_避ける_: ハーネス節（旧規約: CLAUDE.md / AGENTS.md 内の節に記録していた。ADR-0011 で廃止）

## 関係

- **原本** → **生成物** → **管理ブロック** の一方向データフロー。逆流（生成物や管理ブロックの直接編集）はしない
- **共有スキル** は **ユーザースコープ** に属し、install 時に各ツールのスキル置き場へ symlink される
