# imk-harness 用語集

コーディングエージェント用の汎用ハーネス（ユーザー層の設定・知識・規約一式）を管理するリポジトリの用語集。

## 用語

**汎用層** (`generic layer`):
全プロジェクト共通のハーネス資産。このリポジトリが原本を管理し、ホームディレクトリへ展開される。
_避ける_: 共通層

**特化層** (`project-specific layer`):
各プロジェクト側に置くハーネス資産（プロジェクトの CLAUDE.md / AGENTS.md、verify スキルなど）。プロジェクト固有の知識はすべてここに置く。
_避ける_: プロジェクト特化層以外の言い換え（個別層など）

**原本** (`source`):
編集してよい唯一のファイル群（`shared/instructions/*.md`、`shared/skills/*/`）。ここから生成物が作られる。

**生成物** (`generated output`):
`build.sh` が原本から作るファイル（`claude/CLAUDE.md`、`codex/AGENTS.md`）。直接編集禁止。

**管理ブロック** (`managed block`):
ホーム側の CLAUDE.md / AGENTS.md 内でハーネスが所有する、ASCII マーカーで囲まれた区間。区間外はユーザーの自由編集エリアで、絶対に変更しない。
_避ける_: 管理領域、マーカーブロック

**2 段階実行** (`plan-confirm-apply`):
install/uninstall が同一の適用処理を DRYRUN（プラン収集のみ）→ 確認 → 本実行の順で 2 回走らせる方式。
_避ける_: ドライラン方式（DRYRUN は 2 段階実行の前半を指す別概念）

**共有スキル** (`shared skill`):
`shared/skills/` に置かれ、symlink で 3 ツール（Claude Code / Codex / Cursor）に配布されるスキル。どのツールで読まれても本文が成立するように書く。

**ハーネス節** (`harness section`):
プロジェクト CLAUDE.md / AGENTS.md 内の、ハーネス提案の採否を記録する節。「使わない」と記録された項目は再提案しない。

## 関係

- **原本** → **生成物** → **管理ブロック** の一方向データフロー。逆流（生成物や管理ブロックの直接編集）はしない
- **共有スキル** は **汎用層** に属し、install 時に各ツールのスキル置き場へ symlink される

## 解決済みの曖昧さ

- 「層」が 2 つの軸で使われている — 資産の帰属の軸（**汎用層** / **特化層**）と、配置場所の軸（ユーザー層 = ホーム側 / プロジェクト層 = リポジトリ側）。文脈で区別し、帰属の話では「汎用層 / 特化層」を使う（2026-07-13）
